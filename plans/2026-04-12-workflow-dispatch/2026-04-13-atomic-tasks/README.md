Overall, the goal is to create a project and then break it down into smaller, atomic tasks that can be taken to complete the project. The task might be defined at a high level, very vague. In this case, the app should directly ask the user for input, this might be via phone call, or via text message or a series of direct, in app questions.

The workflows should output a set of atomic actions that are necessary to complete a task. Those atomic actions should be prioritizable on a to-do list, so they should be small and self-contained, not large multi-day efforts.

When I create a new project, the workflow should end up creating a set of next actions that need to be taken by me, whether that's review, purchasing things, research, development time, etc.

Where there are unknowns, the app should suggest an amount of time spent clearing unknowns and generating more tasks to do.
